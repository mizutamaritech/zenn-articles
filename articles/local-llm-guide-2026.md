## はじめに

ChatGPTをはじめとするクラウドLLMは便利だが、使い続けるほど気になってくる問題がある。

- **プライバシー**: 入力したテキストはサーバーに送られる。社内情報や個人情報を流せない。
- **コスト**: API利用量が増えると月額が青天井になる。
- **可用性**: サービス障害やAPIの仕様変更に振り回される。
- **検閲**: 倫理フィルターで弾かれ、研究・実験の障害になることがある。

これらを全て解決する選択肢が**ローカルLLM**だ。自分のマシンでモデルを動かし、データは外に出ない。費用は電気代だけ。

ただし「ローカルだから安全」は誤解だ。その誤解を含めて、2026年時点の現実を整理する。

---

## 動作環境の前提（重要）

ローカルLLMの体験はマシンスペックで**劇的に変わる**。

| スペック | 目安 | 動かせるモデル |
|---------|------|--------------|
| VRAM 4GB | エントリー | 3B〜4Bモデル（品質は限定的） |
| VRAM 8GB | **推奨最低ライン** | 7Bモデルが実用的に動く |
| VRAM 16GB以上 | 快適 | 14B〜30Bモデルも可 |
| CPUのみ（RAM 16GB） | 低速だが動く | 7Bモデルを遅く動かせる |

**この記事はVRAM 8GB以上を推奨環境として書いている。**
4GB環境でも動かせるが、品質と速度に妥協が必要になる。

## ローカルLLMとは何か

LLM（大規模言語モデル）とは、大量のテキストで学習した確率的な言語予測モデルだ。
通常はクラウド上の大型サーバーで動くが、**量子化**（精度を落として圧縮する技術）によって、
一般的なGPUでも動かせるサイズまで小さくなった。

```
クラウドLLM:  あなた → API → クラウドサーバー（モデルが動く）→ 応答
ローカルLLM:  あなた → あなたのGPU（モデルが動く）→ 応答
```

データが外に出ない。これがローカルLLMの本質的な価値だ。

## セキュリティの落とし穴

「ローカルだから安全」は半分正解、半分誤解だ。

### 1. 野良モデルのリスク

HuggingFaceには誰でもモデルをアップロードできる。
**悪意あるモデルが存在する可能性がある。**

```bash
# 危険: 出所不明のモデルをそのまま実行する
ollama run unknown-user/suspicious-model

# 安全: 信頼できる提供元のモデルのみ使う
# Google / Alibaba / Meta / Mistral AI などの公式リリースを優先する
ollama run qwen2.5:7b  # Alibaba公式
ollama run gemma3:4b   # Google公式
```

### 2. APIをローカルネットワークに公開するリスク

Ollamaはデフォルトで`localhost:11434`でAPIを提供する。
`0.0.0.0`にバインドし直すと**ネットワーク上の全端末からアクセス可能**になる。

```bash
# 危険: ネットワーク全体に公開
OLLAMA_HOST=0.0.0.0 ollama serve

# 安全: デフォルトのlocalhostのまま使う
ollama serve  # localhost:11434のみ
```

外部から使いたい場合はVPNやSSHトンネル経由にすること。

### 3. モデルの出力を信用しすぎるリスク

ローカルモデルはクラウドLLMより倫理フィルターが薄い場合がある。
**出力をそのまま本番コードや公開文章に使わない。** 必ずレビューを挟む。

## 最小構成セットアップ（Ollama + Qwen2.5）

### Step 1: Ollamaのインストール

```bash
# Linux / macOS
curl -fsSL https://ollama.com/install.sh | sh

# インストール確認
ollama --version
```

:::message alert
インストールスクリプトを実行する前に、URLが`https://ollama.com`であることを確認すること。
フィッシングサイトからの実行は危険だ。
:::

### Step 2: モデルのダウンロードと検証

```bash
# Qwen2.5 7Bをダウンロード（約4.7GB）
ollama pull qwen2.5:7b

# ダウンロードされたモデル一覧を確認
ollama list
```

### Step 3: 動作確認

```bash
# 対話モードで起動
ollama run qwen2.5:7b

# テストプロンプト
>>> 日本語で自己紹介してください
```

レスポンスが返ってきたら成功だ。

### Step 4: APIとして使う（オプション）

```bash
# 別ターミナルでサーバー起動（すでに動いている場合は不要）
ollama serve

# APIリクエスト（localhostのみ）
curl http://localhost:11434/api/generate -d '{
  "model": "qwen2.5:7b",
  "prompt": "ローカルLLMの利点を3つ教えて",
  "stream": false
}'
```
