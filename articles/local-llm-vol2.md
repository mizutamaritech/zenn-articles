まず「WSL2が必要です」ということ。

ほー、WSL2ね。

以前インターネットとかPCって一体何？と思ってラズベリーパイというのを買って意味も分からず何となく触っていた時にたしか入れたな。

何者か調べた。要するに**「Windowsの中にLinuxを住まわせる仕組み」**らしい。

WindowsとLinuxが別の家だとしたら、WSL2はWindowsの家の中に間借りさせるイメージ、と自分なりに解釈した。合ってるかどうかはわからないが、とにかく先に進んだ。

---

## 壁1: Dockerという概念

WSL2を入れたら次はDockerだった。これもラズベリーパイでセキュリティって何？と思った時に出てきたなと薄っすらした記憶を辿る。

「コンテナ」という言葉が出てきた。コンテナ。船に積む金属の箱のやつ？

調べると「アプリをまるごと箱に詰めて動かす技術」とある。

つまり、Linuxの家の中に、さらに箱を置いて、その中でOpenclawを動かす。

二重構造だ。なぜそんな複雑なことを、と思ったが、後でその意味がわかる。箱の中で動いているから、何かあっても外（自分のPC）には影響が出ない。隔離がセキュリティになるのだと。だけど100％のセキュリティなどありえないという意識は必要だ。

```bash
docker ps
```

このコマンドを打って、動いているコンテナの一覧が表示されたとき、少しだけ霧が晴れた気がした。

---

## 壁2: 設定ファイルとの格闘

次は設定ファイルだった。`.env`というファイルにAPIキーを書く。

```
ANTHROPIC_API_KEY=ここに貼る
DISCORD_BOT_TOKEN=ここに貼る
```

単純なことに見える。でも実際にやると、スペースが余分に入っていたり、引用符の有無で動かなかったり。

エラーが出るたびに意味がわからなかった。ターミナルに赤い文字が並ぶ。何が悪いのか、どこを直せばいいのか。

そこで助けてもらったのが、まさにAIだった。エラー文をそのまま貼り付けて「これどういう意味」と聞く。するとどこが間違っているか教えてくれる。エラーと対話しながら進むという感覚が、少しずつ掴めてきた。けど完全に理解する事はできないので触りながら覚えていくスタイルで進む。

`openclaw.json`のJSON構文エラーで詰まったときは、カンマが一つ余分なだけだった。人間の目には見えない。でもコンピュータは正直だ。
細かい所まで気にする。こんな人間がいたらちょっと避けたいなと思った。
---

## 壁3: セキュリティという意識

設定ファイルを触っていると、こんな記述があった。

「.envはGit管理外にすること」

最初、意味がわからなかった。Gitって何を管理してるの？

調べると、`.env`にはAPIキーが書いてある。GitHubに上げてしまうと、世界中の人に見られる。つまり、自分のAPIキーが公開される。

——怖い。万が一という事もあるのでさらに追加のセキュリティとしてAPIキーの利用する金額の上限も設定しておいた。

それまで漠然とやっていたが、このとき初めてセキュリティを自分ごととして感じた。ファイルの権限を600に設定するとか、GitHubに何を上げて何を上げないか、意識するようになったのはここからだ。

全部わかったわけじゃない。でも「わからないまま進む」と「わからないけど意識する」は違う。

## そして、動いた

```bash
docker compose up -d
```

このコマンドを打って、しばらく待った。

Discordを開いた。テキストを打った。

返事が来た。

vol.1で書いた感動が、ここで戻ってきた。でも今回は少し違う。仕組みがうっすら見えた上での感動だった。
初めて動かしたときは新たな生命の誕生を見た気がしたが、ここはアラジンと魔法のランプのランプを擦ってジーニーを呼び出した感覚に近い。

WSL2があって、Dockerがあって、設定ファイルがあって、その上でAIが動いている。全部自分の手で繋いだ。

完全に理解はしていない。でも「自分が動かしている」という実感は、本物だった。

## 全自動ではなく、許可を得ながら

今、Openclawと一緒にこの記事を書いている。

Openclawは提案する。私が判断して許可する。Openclawが実行する。

全自動じゃない。でも今はそれでいいと思っている。今の私には、何が起きているかを確認しながら進む方が安心だし、学べる。

もう少し理解が深まったら、もっと任せられるかもしれない。全自動で記事が公開される未来も、あるかもしれない。でも今は、人間が確認するというステップがセキュリティでもあり、学びでもある。
